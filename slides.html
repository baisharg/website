21K tokens ≈ 84K chars ≈ 2,000 líneas de código

💡 Por qué BMad es Diferente

Los expertos fueron más lentos porque dieron a la AI TODO el contexto (codebase completo, 60K+ tokens).

BMad agents reciben SOLO contexto focalizado (un epic, arquitectura relevante, <5K tokens).

No es sobre tener AI, es sobre dar a la AI la cantidad CORRECTA de contexto.

🧠 El Problema Posta: Context Window

📉 El modelo pierde precisión con mucho contexto

(LoCoDiff Benchmark - Sonnet 4.5)
• 2-21K tokens → ~96% precisión
• 21-35K tokens → ~82% precisión
• 36-60K tokens → ~72% precisión
• 60-98K tokens → ~64% precisión

🎯 Se pierde lo importante

El modelo no encuentra los detalles que importan cuando hay mucho texto

♻️ Un solo chat = El sistema se deteriora completamente

Cada cambio agrega basura hasta que explota todo

✅ La que va: Manejo de Contexto Inteligente

Agentes especializados que buscan SOLO lo que necesitás para cada tarea

🤖 ¿Qué es un Agente BMad?

💬 Chatbot Tradicional (ChatGPT/Claude)

  • • Un solo chat con TODO el contexto
  • • Recibe tu codebase completo
  • • Se pierde entre 60K+ tokens
  • • No tiene rol específico
  • • Trata de hacer todo a la vez

⚡ Agente BMad

  • • Especializado: Un rol claro (PM, Dev, QA)
  • • Contexto focalizado: Solo lee lo que necesita
  • • Bajo tokens: <5K tokens por tarea
  • • Experticia: Prompt optimizado para su rol
  • • Coordinado: Trabaja con otros agentes
Técnicamente:

Un agente = Mismo LLM (Claude/GPT) + Prompt especializado + Contexto específico

Ejemplo: PM Agent lee requirements.md (2K tokens) → Architect Agent lee PRD + tech-stack.md (3K tokens) → Developer lee epic-1.md + architecture-backend.md (1.5K tokens)

🖥️ Dónde usás los agentes:

// src/controllers/taskController.ts
export async function createTask(req: Request, res: Response) {
  const { title, description, priority } = req.body;

  // Validation
  if (!title || title.length < 1) {
    return res.status(400).json({ error: 'Title required' });
  }

  // Create task
  const task = await prisma.task.create({
    data: { title, description, priority, userId: req.user.id }
  });

  return res.status(201).json(task);
}

// Tests included ✓

🔌 MCP Tools: Superpoderes para tus agentes

📖 ¿Qué es MCP?

MCP = Model Context Protocol - Un estándar para darle "herramientas" a los LLMs.
Los agentes pueden controlar un browser, ejecutar código, hacer búsquedas, etc.

🎭 Playwright MCP - Browser Automation

Los agentes pueden navegar y testear tu app automáticamente en el browser.

✨ Capacidades: Navegar, clicks, llenar forms, screenshots, E2E testing
🎯 Casos de uso: QA testing, responsive design, user flows, visual bugs

🧠 Sequential Thinking MCP - Problem Solving

Estructurado paso a paso para resolver problemas complejos. Los agentes pueden revisar, refinar y ajustar su razonamiento dinámicamente.

✨ Features: Break down problems, revise thoughts, branch reasoning, verify hypotheses
🎯 Ideal para: Planning, design, debugging complejo, análisis que necesita revisión
